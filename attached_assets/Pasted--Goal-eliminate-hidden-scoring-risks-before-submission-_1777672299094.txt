# 1️⃣ ADD QUALITATIVE MESSAGE ASSERTIONS

Extend existing tests beyond structural checks.

## Add the following validations for EVERY generated message:

### 1. Trigger grounding check

Ensure message explicitly references trigger context.

Examples:

* perf_dip → must include words like "dropped", "declined", "vs baseline"
* renewal_due → must include "expires", "days"
* research_digest → must include "trial", "study", "issue"

```python

```

---

### 2. Causal structure detection

Ensure message follows:
problem → proof → action

Heuristic:

* contains decline/problem word
* contains number
* contains action phrase

```python

```

---

### 3. Compulsion enforcement

Ensure at least ONE of:

* urgency: "today", "tonight", "days", "deadline"
* social proof: "nearby", "similar", "peer"
* loss: "dropped", "lost", "lapsed"
* reciprocity: "already drafted", "ready in"
* curiosity: "spotted", "interesting"

```python
assert any(word in body.lower() for word in compulsion_keywords)
```

---

### 4. Ban generic phrases

Reject weak content:

```python

  "boost sales",
  "improve growth",
  "let me know",
  "increase visibility"

```

### 5. Merchant grounding

Ensure at least ONE:

* merchant name

* offer
